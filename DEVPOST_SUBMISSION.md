# BOB — A Production-Ready Agent Harness for Browser Customization

**Track: Augment the Agent**

> Naked LLMs hallucinate selectors, blow past CSP, and write code that
> works once and breaks on the next deploy. BOB is the runtime
> scaffolding that turns a stateless model call into a reliable,
> verifiable, user-facing browser agent — and ships every pattern
> developers need to build their own.

---

## The problem we solve (real-world toil)

Anyone who has tried to point a coding agent at a live webpage knows the
failure modes:

- The model hallucinates a selector that hasn't existed since the last
  React deploy.
- It writes `innerHTML = ...` and crashes on every site with a strict
  Trusted Types policy (YouTube, Google, Twitter).
- It "succeeds" without ever running the code, so silent failures pile up.
- It produces a one-shot script with no plan for re-applying when the SPA
  navigates or the DOM mutates.
- It can't ask the user *"is this what you wanted?"* — so the user is
  stuck with a black-box install or a manual code review they can't read.

Every one of these is a generic agent-reliability problem: bad context,
no verification, no human-in-the-loop, no extensibility. **BOB is the
harness that closes those gaps for browser-acting agents — and the
patterns are reusable for any agent that touches a live, mutating
environment.**

---

## What BOB is

A Chrome extension (Manifest V3, ~6,000 lines of TypeScript) that lets
any user press **⌘K** on any webpage, describe a customization in plain
English, and get a working content-script feature installed in under 10
seconds. The extension is provider-agnostic — it ships a unified
[`Provider`](src/background/providers/types.ts) interface with
implementations for Anthropic Claude, OpenAI GPT, and Google Gemini /
Gemma — so the same harness benefits any frontier model the user prefers
to bring.

BOB is **bring-your-own-key**, runs entirely on-device
(`chrome.storage.local`), and never proxies prompts. That is not just a
privacy story — it is the deployment model that makes this realistic for
daily use today.

## How BOB hits the five "Augment the Agent" pillars

### 1. Better verification for AI outputs

Naked LLM output is unreliable. BOB layers four independent verification
mechanisms:

- **`test_code` tool** ([tools.ts:66-104](src/background/tools.ts#L66-L104)).
  Before committing a final answer, the agent can execute candidate JS in
  the user's actual tab and read back DOM-delta + thrown error. Verified
  against ground truth, not the model's confidence.
- **Reflexion retry loop**
  ([content/index.ts:101-160](src/content/index.ts#L101-L160)). When
  installed code throws, BOB feeds the runtime error and a fresh DOM
  snapshot back to the agent and asks it to fix the *root cause* — up to
  two retries. This is Reflexion-pattern in production, not in a paper.
- **Idempotency contract** baked into the system prompt
  ([prompt.ts:33-41](src/background/providers/prompt.ts#L33-L41)). Every
  generated mutation is tagged `data-bob='<slug>'`, every observer is
  keyed by slug — so re-running the same code doesn't compound side
  effects. This makes verification *stable* across SPA navigations and
  toggle cycles.
- **Strict output contract** ([agent.ts:165-211](src/background/agent.ts#L165-L211)).
  The final response is parsed through three fallbacks (raw JSON,
  fenced, balanced-brace extraction) before it touches storage. Models
  that ramble get caught before they install anything.

### 2. Smarter context retrieval

A 2 MB DOM dump destroys both the cost budget and the model's attention.
BOB's context pipeline is a pragmatic case study:

- **`prunePage`** ([dom-prune.ts](src/content/dom-prune.ts)) — a
  synchronous, non-throwing renderer that produces a ≤4 KB text snapshot
  under a 50 ms wall budget. It strips autogenerated class names
  (`css-AbCdEf`, `jsx-12345`), prioritizes stable identifiers
  (`id` > `data-testid` > `aria-label` > stable class), and collapses
  long sibling runs. Generic enough that any browser-acting agent can
  adopt it as-is.
- **`query_dom` tool** ([tools.ts:36-64](src/background/tools.ts#L36-L64)).
  When the snapshot isn't enough, the agent issues targeted CSS-selector
  probes against the live page and gets back element descriptors. Tool
  use replaces speculative pattern-matching.
- **Refinement history with capping**
  ([overlay.ts:283-294](src/content/overlay/overlay.ts#L283-L294)). Past
  assistant turns carry a one-line *summary* of what was built, not the
  full code, so the agent sees intent without re-ingesting prior outputs
  verbatim. Last 6 turns retained — bounded prompt growth without
  losing thread.
- **Behavior-driven retrieval**
  ([behavior-tracker.ts](src/content/behavior-tracker.ts) +
  [suggestions-engine.ts](src/background/suggestions-engine.ts)). BOB
  watches the user's clicks (privacy-bounded — no input/contenteditable/
  password reads, sensitive hostnames skipped) and *retrieves the right
  agent action at the right time* — proposing "auto-dismiss this banner"
  the moment the user has dismissed it three times by hand.

### 3. Agent integrations & extensions

BOB is, by construction, an **extensible agent runtime**:

- **Three frontier providers, one interface** —
  [`anthropic.ts`](src/background/providers/anthropic.ts),
  [`openai.ts`](src/background/providers/openai.ts),
  [`google.ts`](src/background/providers/google.ts) all conform to the
  same `Provider.chat({ messages, system, tools, apiKey, model, effortMode })`
  contract. Adding a fourth (xAI, Mistral, local Ollama) is one file.
- **Effort-mode normalization** is the cleanest part of the abstraction.
  A single `effortMode: 'high'` flag maps to Anthropic extended thinking
  (`thinking.budget_tokens=8000`,
  [anthropic.ts:84-96](src/background/providers/anthropic.ts#L84-L96)),
  OpenAI `reasoning_effort: 'high'` on gpt-5 reasoning models
  ([openai.ts:84-89](src/background/providers/openai.ts#L84-L89)), and
  Gemini `thinkingBudget` / Gemma `thinkingLevel`
  ([google.ts:99-108](src/background/providers/google.ts#L99-L108)).
  Switching providers does not regress reasoning capability.
- **Pluggable tool registry** at
  [tools.ts:4-29](src/background/tools.ts#L4-L29). Adding a tool is two
  steps: declare its schema, append to `dispatchTool`. The agent loop
  ([agent.ts:118-158](src/background/agent.ts#L118-L158)) doesn't change.
- **Browser-native distribution**: Manifest V3 service worker +
  `chrome.scripting.executeScript` with on-the-fly Trusted Types policy
  creation ([index.ts:81-95](src/background/index.ts#L81-L95)). This is
  the integration that lets generated code run on sites — like
  YouTube — that reject every other injection method.
- **Import / export** ([import-export.ts](src/popup/import-export.ts)) —
  features serialize to portable JSON. Teams can ship curated feature
  packs; users can hand off agent-generated tooling to teammates.

### 4. Human–AI collaboration tooling

Where most agent demos are fire-and-forget, BOB treats the user as an
equal participant in the loop:

- **Preview-before-install gate**
  ([overlay.ts:307-322](src/content/overlay/overlay.ts#L307-L322)).
  Generated code is never auto-applied. The user sees name, description,
  URL pattern, and the code itself in a syntax-aware preview, edits the
  metadata, and explicitly clicks Install.
- **Conversational refinement** ([overlay.ts:266-322](src/content/overlay/overlay.ts#L266-L322)).
  After install, the overlay flips into refine mode — *"smaller", "apply
  more broadly", "undo this"*. The agent is given the full installed
  code plus prior turns, so refinement is a real conversation rather
  than a re-prompt.
- **Edit-with-context** ([overlay.ts:853-874](src/content/overlay/overlay.ts#L853-L874)).
  Re-opening any installed feature seeds the prompt, the prior code, and
  the iteration tree (`parentFeatureId`, `iterationNumber`) so versions
  are linked, not lost.
- **Three-state suggestion dismissal** —
  Try / Later (3-day cooldown) / Never
  ([suggestions-engine.ts:283-311](src/background/suggestions-engine.ts#L283-L311)).
  This is the right primitive for proactive agents: the user can defer
  without permanently rejecting, and BOB respects that without
  re-pestering.
- **Voice input** ([voice-input.ts](src/content/voice-input.ts)) — Web
  Speech API on the prompt textarea. Turns the agent into something a
  user can drive while their hands are on something else.
- **Surfaced failures** — every feature carries `runCount`, `errorCount`,
  `lastError`; the popup expands the actual stack trace with copy-to-
  clipboard ([popup.ts:416-440](src/popup/popup.ts#L416-L440)). When the
  agent is wrong, the user knows immediately and can correct it.

### 5. Eliminating professional toil

The toil this removes is concrete and measurable:

- **Userscript engineering, gone**. A workflow that previously meant
  "open DevTools, write a Tampermonkey script, debug your selectors,
  redeploy when the site changes" collapses to a ⌘K prompt. We
  preserved that workflow's power (full content-script JS, MAIN-world
  injection, mutation observation) while removing the code itself.
- **No copy-paste between tools.** Most agent products force the user to
  copy generated code into a separate runtime. BOB *is* the runtime —
  generation, preview, installation, monitoring all in one shadow-DOM
  overlay.
- **No deploy step.** The feature ships immediately and runs on every
  matching URL, including SPA navigations
  ([spa.ts](src/content/spa.ts) + the History API patch installed via
  background-side `chrome.scripting.executeScript`,
  [index.ts:155-191](src/background/index.ts#L155-L191)).
- **Proactive automation discovery.** Most "automate the boring stuff"
  tooling makes the user *notice* the boring stuff first. BOB's
  behavior tracker does that for them — three repeated dismissals on a
  hostname surface a "we noticed you keep dismissing this — want it
  gone?" suggestion. The agent meets the user before the user meets the
  problem.
- **Operator cost: $0.** BYOK + on-device storage means there is no
  server we have to host, scale, secure, or charge for. This is the
  difference between a hackathon demo and something a team installs
  Monday morning.

## Architectural decisions that prove this is production, not demo

- **Per-key storage locks** ([storage.ts:10-20](src/shared/storage.ts#L10-L20)).
  `chrome.storage.local` has no CAS, so two concurrent tabs racing on
  RECORD_FEATURE_RESULT would clobber each other. We serialize per-key.
  This is the kind of bug that bites you on day 30, not day 1.
- **Trusted Types policy created lazily**
  ([index.ts:81-95](src/background/index.ts#L81-L95)). Each injection
  creates a unique policy name; we fall back to the default policy if
  one exists. Works on every CSP-strict site we tested without hard-
  coding allowlists.
- **`__bobObserve` reactivity helper** installed in MAIN world from the
  background ([index.ts:192-247](src/background/index.ts#L192-L247)).
  Generated code calls `window.__bobObserve(cb, { slug })` and gets a
  debounced, deduped, slug-keyed MutationObserver — so the agent's
  generated code stays simple and idempotent across SPA navigation,
  toggle-off-toggle-on, and infinite scroll.
- **Privacy as a hard constraint, not a setting.** The behavior tracker
  refuses to attach listeners on hostnames matching `bank|auth|login`
  ([behavior-tracker.ts:225-232](src/content/behavior-tracker.ts#L225-L232))
  and walks ancestors to skip clicks inside any input / textarea /
  contenteditable / password subtree. This is the level of care that
  makes the difference between "I'd never run this" and "fine, install it."
- **Iteration-cap-by-effort-mode** ([agent.ts:90-107](src/background/agent.ts#L90-L107)).
  Standard runs cap at 6 tool turns; high-effort doubles to 12 to give
  extended-thinking models room to plan. Caps are explicit, not
  implicit, which is what production agents need.

## Why this wins on "Augment the Agent"

The track asks for tools real teams would use daily. BOB is one we use
every day already:

1. **Verification, retrieval, integration, collaboration, toil-reduction
   are not slides — they are modules in this repo.** Every pillar maps to
   files we can open in the demo.
2. **The patterns generalize.** The DOM pruner, reflexion controller,
   tool dispatcher, three-state dismissal model, and provider-effort-
   normalization are extractable building blocks for any team building
   browser agents — coding agents that need to test on real sites,
   research agents that need to scrape modern SPAs, accessibility agents
   that need to rewrite live UIs.
3. **It runs today, on every Chromium browser, with the model the user
   already pays for.** No backend, no rollout plan, no waitlist.

We didn't build an agent demo. We built the runtime an agent needs to be
useful on the live web — and we shipped it as a tool a non-developer can
use in 30 seconds.

That is the augmentation.

## Try it in 90 seconds

```bash
npm install && npm run build
# Load dist/ as an unpacked extension at chrome://extensions
# Right-click the icon → Options, paste an Anthropic / OpenAI / Gemini key
# Press ⌘K on any webpage. Type what you want.
```

Demo seeds (YouTube Shorts hider, HN star titles, Reddit ad blocker,
Wikipedia dim sidebar, example.com tint) are loadable from the Options
page with one click — perfect for a clean stage demo.
